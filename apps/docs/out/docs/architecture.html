Architecture

System design, data model, and request flow

System Overview

PolicyFlow is a single Go binary that embeds the compiled Next.js frontend using go:embed. There are no external service dependencies — only a local SQLite file.

Data Model

Policy State Machine

Transitions are enforced in PUT /api/policies/:id. The status field accepts: Draft, Review, Published, Archived.

Future Integration Points

PolicyFlow is designed to be extended:

ExtensionHook Point
SSO / OIDCReplace magic-link handler in internal/handlers/auth.go
Employee importNew POST /api/users/import handler reading CSV/SCIM
LDAP syncBackground goroutine syncing users from LDAP directory
Department scopingAdd allowed_departments to policies; filter in ListPolicies
WebhooksFire events from acknowledgement/publish handlers
